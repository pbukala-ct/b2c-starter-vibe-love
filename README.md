# B2C Starter — commercetools + Claude Code

A fully functional B2C ecommerce storefront built with **Next.js 16**, **TypeScript**, **Tailwind CSS v4**, and **commercetools**. Every line of code — the storefront, admin tools, and tests — was generated by [Claude Code](https://docs.anthropic.com/en/docs/claude-code) from plain-English instructions.

Fork it, connect your own commercetools project, and use Claude Code to customize it into your own demo.

---

## What You Get

- **Product catalog** with category browsing, mega menu, and full-text search with filters
- **Product detail pages** with variant selection, image gallery, and Subscribe & Save
- **Cart and checkout** — guest or logged-in, split shipments, credit card with test auto-fill
- **Customer accounts** — profile, order history, subscriptions (pause/resume/skip), address book, payment methods
- **Internationalization** — country/currency/language switching (US, GB, DE)
- **Admin tools** (`tools/`) for catalog exploration, subscription setup, and more
- **E2E tests** generated from plain-English descriptions in `test.txt`

See **[FEATURES.md](FEATURES.md)** for the complete feature inventory with implementation details.

## Quick Start

### 1. Fork This Repo

1. Click the **Fork** button at the top-right of this page
2. Set the **Owner** to the `commercetools-demo` org
3. Give it a name that fits your demo (e.g., `acme-storefront`)
4. Click **Create fork**

Then clone your fork:

```bash
git clone https://github.com/commercetools-demo/YOUR-REPO-NAME.git
cd YOUR-REPO-NAME
```

### 2. Set Up commercetools Credentials

You need two API clients in your CT Merchant Center (**Settings > Developer > API Clients**):

**Storefront client** (`site/.env`):
- Use the **Frontend B2C** template
- Add scopes: `manage_payments`, `manage_recurring_orders`, `manage_custom_objects`
- Download the .env file to `site/.env`
- Open Claude Code in the project and ask: *"Generate a SESSION_SECRET and add it to site/.env"*

**Admin client** (`tools/.env`) — optional, for admin tools:
- Use the **Admin client** template
- Download the .env file to `tools/.env`

> **Warning:** `.env` files contain secrets and are gitignored. **Never commit them.** If you accidentally commit a `.env` file, rotate your API client credentials immediately in Merchant Center.

### 3. Install and Run

```bash
cd site
npm install
npm run dev -- -p 8888
```

Open http://localhost:8888

## Deploy to Netlify

`netlify.toml` is pre-configured. To deploy:

1. Push your fork to GitHub
2. In [Netlify](https://app.netlify.com), click **Add new project > Import a Git repository > GitHub**
3. Connect your GitHub repo — build settings are auto-detected from `netlify.toml`
4. Add your `site/.env` variables in **Site configuration > Environment variables**
5. Deploy

Every push to **main** triggers an automatic redeploy.

### Required Environment Variables

| Variable | Description |
|---|---|
| `CTP_PROJECT_KEY` | commercetools project key |
| `CTP_CLIENT_ID` | Storefront API client ID |
| `CTP_CLIENT_SECRET` | Storefront API client secret |
| `CTP_AUTH_URL` | CT auth URL |
| `CTP_API_URL` | CT API URL |
| `CTP_SCOPES` | API client scopes |
| `SESSION_SECRET` | Random string for JWT signing (ask Claude Code to generate one) |

## Using Claude Code

This project includes a `CLAUDE.md` file that gives Claude Code full context about the architecture, patterns, and conventions. Open the repo in Claude Code and start asking for changes:

- *"Add a wishlist feature"*
- *"Change the color scheme to match our brand"*
- *"Add product reviews to the product detail page"*
- *"Write a tool that creates 50 test products in my CT project"*
- *"Deploy to Netlify"*

Claude Code will explore the codebase, understand the patterns, make changes, and verify them.

### The `tools/` Directory

Instead of wiring up MCP servers or writing API calls by hand, just ask Claude Code:

> *"Write a tool that lists all products without images and put it in tools/"*

It follows the existing pattern — imports from `ct-admin.mjs`, uses the admin `.env`, and produces a standalone script you can run anytime.

### Adding Features Without Breaking Things

Three files work together as guardrails:

| File | Purpose |
|---|---|
| `CLAUDE.md` | Tells Claude Code the architecture rules and conventions |
| `FEATURES.md` | Complete inventory of what's implemented — Claude Code reads this before making changes |
| `test.txt` | Plain-English test descriptions that generate Playwright E2E tests |

**Recommended workflow:**

1. Tell Claude Code what you want — *"Add a wishlist feature"*
2. Claude Code reads `CLAUDE.md` and `FEATURES.md` automatically, so it knows the architecture and what exists
3. Claude Code implements the feature
4. Claude Code updates `FEATURES.md` and adds test descriptions to `test.txt`
5. Claude Code regenerates and runs all tests — existing tests catch regressions, new tests validate the feature

You can also ask Claude Code to run just the tests at any time: *"Run the tests against localhost"*

## Project Structure

```
├── site/               # Next.js storefront
│   ├── app/            #   App Router pages and API routes
│   ├── components/     #   React components
│   ├── lib/            #   Shared utilities and CT client
│   ├── context/        #   React context providers
│   └── hooks/          #   Custom React hooks
├── test/               # Playwright E2E tests
├── tools/              # Admin scripts for CT setup
├── CLAUDE.md           # Context file for Claude Code
├── FEATURES.md         # Complete feature inventory
├── initial_prompt.txt  # The original prompt that generated the site
├── test.txt            # Plain-English test descriptions
├── netlify.toml        # Netlify build configuration
└── .claude/            # Claude Code dev server config
```

## Running Tests

```bash
cd test
npm install
npx playwright install --with-deps chromium
npx playwright test
```

Tests run against the live Netlify deployment by default (`BASE_URL` in `test/playwright.config.ts`). To test locally:

```bash
BASE_URL=http://localhost:8888 npx playwright test
```
